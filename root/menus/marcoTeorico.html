Marco Teórico

A lo largo de esta sección, exploraremos las bases teóricas que sustentan el Machine Learning y que son esenciales para comprender cómo las máquinas pueden aprender de los datos y resolver problemas de manera autónoma.

¿Qué es Machine Learning?

En primer lugar, es esencial comprender qué es Machine Learning. Este es una rama de la inteligencia artificial que se centra en desarrollar algoritmos y modelos que permiten a las computadoras aprender y mejorar su rendimiento en tareas específicas a través de la experiencia y los datos, en lugar de ser programadas explícitamente. Esencialmente, se trata de enseñar a las máquinas a aprender de manera autónoma y tomar decisiones basadas en patrones y conocimiento extraído de los datos. El Machine Learning tiene una amplia variedad de aplicaciones en campos como la predicción, la clasificación, el reconocimiento de patrones y la toma de decisiones automatizada.

Metodología de trabajo

Proceso CRISP-DM

En el corazón de cualquier proyecto de Machine Learning se encuentra el proceso CRISP-DM (Cross-Industry Standard Process for Data Mining).
Es una metodología estándar ampliamente reconocida para la gestión de proyectos de minería de datos y Machine Learning. CRISP-DM consta de seis fases clave que se ejecutan en secuencia, pero también pueden ser iterativas según las necesidades del proyecto:


Comprensión del negocio: En esta fase inicial, se trabaja en estrecha colaboración con los stakeholders para comprender los objetivos del proyecto, los requisitos del negocio y cómo la minería de datos puede aportar valor.
Comprensión de los datos: Aquí se adquiere una comprensión profunda de los datos disponibles, su calidad y su relevancia para el proyecto. Esto incluye la recopilación y evaluación de los datos, así como la identificación de posibles problemas.
Preparación de datos: En esta fase, los datos se limpian, transforman y preparan para su uso en la construcción de modelos. Esto implica manejar valores faltantes, eliminar duplicados y realizar la codificación de variables, entre otras tareas.
Modelado: Se seleccionan y se construyen modelos de Machine Learning en esta etapa. Se prueban varios algoritmos y técnicas, y se ajustan para lograr un rendimiento óptimo. Es una fase crucial para la creación de modelos efectivos.
Evaluación: En esta fase, se evalúan los modelos desarrollados para determinar su calidad y eficacia. Se utilizan métricas como la precisión, el F1-score y la matriz de confusión para evaluar el rendimiento de los modelos.
Despliegue: Una vez que se ha seleccionado el modelo final y se ha demostrado su eficacia, se procede a su implementación en el entorno operativo del negocio. Esto permite que el modelo se utilice para tomar decisiones basadas en datos en la práctica.

CRISP-DM es una metodología flexible que puede adaptarse a una variedad de proyectos de minería de datos y Machine Learning. Proporciona una estructura organizada que facilita la gestión de proyectos, desde la fase de exploración inicial hasta la implementación exitosa de modelos en la operación del negocio. Es ampliamente utilizada y respetada en la comunidad de análisis de datos y aprendizaje automático.

Semma

Similar a CRISP-DM, SEMMA (Sample, Explore, Modify, Model, Assess) es una metodología desarrollada por SAS que consta de cinco etapas: Muestreo, Exploración, Modificación, Modelado y Evaluación. Se centra en el análisis de datos y el modelado estadístico.

KDD

KDD (Knowledge Discovery in Databases) se enfoca en la extracción de conocimiento útil a partir de grandes conjuntos de datos. Abarca tareas desde la selección de datos hasta la visualización de resultados y es utilizado en minería de datos y ML.

AgileAI

AgileAI combina principios de metodologías ágiles con enfoques específicos para proyectos de IA y ML, promoviendo la colaboración y la iteración continua.

Tratamiento previo de datos

El tratamiento previo de datos en Machine Learning es una etapa esencial que implica la preparación y limpieza de los datos antes de utilizarlos para entrenar modelos. Las actividades clave en esta fase son:
1. Recopilación de datos: Se adquieren los datos necesarios de diversas fuentes, como bases de datos, archivos, sensores, o fuentes en línea.
2. Exploración de datos: Se realiza un análisis inicial para entender la estructura de los datos, identificar valores faltantes o atípicos, y determinar la distribución de las variables.
3. Limpieza de datos: Se eliminan o se llenan los valores faltantes, y se manejan los datos atípicos de acuerdo con la estrategia seleccionada. Esto asegura que los datos sean coherentes y confiables.
4. Transformación de datos: Se aplican transformaciones a los datos, como normalización, escalado o codificación de variables categóricas, para que sean adecuados para el entrenamiento de modelos.
5. Selección de características: Se eligen las características más relevantes para el problema y se descartan las que aportan poca información. Esto puede mejorar la eficiencia del modelo y reducir el riesgo de sobreajuste.
6. Ingeniería de características: Se crean nuevas características a partir de las existentes para capturar mejor patrones y relaciones en los datos.
7. División de datos: Se divide el conjunto de datos en conjuntos de entrenamiento, validación y prueba para evaluar el rendimiento del modelo de manera adecuada.
8. Balanceo de clases: En problemas de clasificación desequilibrados, se pueden aplicar técnicas para equilibrar las clases y evitar sesgos en el modelo.
9. Validación cruzada: Se utiliza validación cruzada para evaluar el rendimiento del modelo de manera más robusta y evitar el sobreajuste.
10. Gestión de datos sensibles: En proyectos con datos sensibles, se implementan medidas de seguridad y privacidad para proteger la información confidencial.

El tratamiento previo de datos es fundamental para garantizar la calidad de los datos y la eficacia de los modelos de Machine Learning. Un conjunto de datos bien preparado es esencial para obtener resultados precisos y confiables en la fase de modelado y evaluación.

Tipos de aprendizaje

Aprendizaje supervisado

Los algoritmos supervisados son utilizados en tareas de aprendizaje donde se dispone de un conjunto de datos etiquetado, es decir, cada entrada del conjunto de datos se encuentra asociada con una etiqueta o un resultado conocido. El objetivo de los algoritmos supervisados es aprender una función que mapee las características de entrada a las etiquetas de salida, de modo que el modelo pueda hacer predicciones precisas sobre datos nuevos y no vistos.

Aprendizaje no supervisado

Los algoritmos no supervisados, en cambio, se utilizan en situaciones en las que los datos no están etiquetados, y el objetivo es encontrar patrones o estructuras intrínsecas en los datos sin una guía explícita. Estos algoritmos se utilizan principalmente en tareas de agrupación (clustering) y reducción de dimensionalidad.
